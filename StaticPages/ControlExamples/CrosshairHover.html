Filtering the crosshair separately by row and column

Note that when both the rowSelector and columnSelector have values, the crosshairSelector will only affect the currently rolled-over cell.

$('#headersAndFootersOnly').crosshairHover({ crosshairSelector: 'tbody th, thead th, tfoot td:not(:first-child)', crosshairRowSelector: 'tbody th', crosshairColumnSelector: 'thead th, tfoot td:not(:first-child)' });

Mix and match Crosshair Hovers

/* column header - highlight only the data in the column and footer, without the header */ $('#multiHover').crosshairHover({ hoverSelector: 'thead th', crosshairSelector: 'tbody td, tfoot td' }); /* row header - highlight the entire row, but don't do the "cross" column banding */ $('#multiHover').crosshairHover({ hoverSelector: 'tbody th', crosshairColumnSelector: ' ' /* space means: never highlight the column elements */ }); /* body - highlight the header and footer cells only, and don't highlight the hovered-over cell */ $('#multiHover').crosshairHover({ hoverSelector: 'tbody td', crosshairSelector: ' ', /* space means: never highlight the rolled-over element */ crosshairRowSelector: 'th', crosshairColumnSelector: 'th, tfoot td' }); /* footer - highlight the data part of the footer, and the column, using a special class */ $('#multiHover').crosshairHover({ hoverSelector: 'tfoot td:not(:first-child)', crosshairClass: 'importantHover' });

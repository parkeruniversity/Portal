Setup Steps for EX

In order to successfully implement Staff, the following steps should be performed in the order in which they are listed.

NOTE:  Your JICS Administrator has already defined roles that have access to individual portlets via the Allow Access check box (Site Manager link, Global Portlet Operations Edit link).  For more information about the Allow Access global operation, see the Getting Started with JICS.

Step 1:  Assign Employees to Staff Administrator and Staff Roles

Select the individuals who will have staff administrator or staff privileges and add that role to their ID number using EX software.

  • Go to the Maintain Web Access Code window (EX, Web Administration, Task, Web Administration, Access/Permissions, Maintain Web Access Code).

    • Right-click the lower gray area of the window and select add row.  

    • Enter the ID number of the individual you wish to grant access to the Staff CRM.  

    • In the Access Code column, enter the password this individual will use to log in to JICS.

    • Save.

  • Go to the Group Membership window (EX, Web Administration, Task, Web Administration, Access/Permissions, Group Membership).

    • Right-click the lower gray area of the window and select add row.  

    • Enter the ID number of the individual you gave an access code to on the Maintain Web Access Code window.

    • Select the JICS role (either Staff or Staff Administrator) from the Group ID column.

    • Save.

Note: The JICS User Name is the ID Number used in EX, and the Password is the Access Code saved on the Maintain Web Access Code window.

Step 2:  Create and Assign users to Groups

  • Use the Task List module to define staff members as users (EX, Task List, TaskList, User).

    • Right-click in the window and select Add Row.

    • Enter a User ID and Description to identify this individual as a user.

    • Enter the individual's ID Number.

    • Save.

  • Then, use the Task List module to associate these users with at least one group per user (EX, Task List, TaskList, Groups).  Consider your accounting structure when creating groups as these groups will be used by the GL Account Lookup and Requisition Entry portlets when granting account security access (refer to steps 3 and 4 below for setup instructions).  

    • Right-click in the window and select Add Row.

    • Enter a Group ID and Description to create a new group.

    • Select the new row and click the Membership button.  

    • Add members to the group by moving those listed as non-members to the members column.  Don't forget to add your new user.

    • Click OK.

Step 3:  Set up General Ledger (GL) Account Security

Set up account-level security for the groups created in step 2.  This setup will be important for portlets that access account information.  

  • You can do this on the G/L Security window of the General Ledger module (EX, Business Office, Admin, Task, Configure Business Office, G/L Security...).

    • For more information about this window, click F1 to access the G/L Security window Help topic.

Step 4:  Set up Requisition Entry Defaults

The Requisition Entry portlet enables Staff to enter requisitions via the Web.  Staff can also assign groups, created in step 2, to approval tracks and identify the account numbers to bill each group when requisitions become purchase orders.

  • Use the Requisition Entry Defaults window of the Purchasing module to assign the group to an approval track (EX, Business Office, Purchasing, Admin, Task, Configure Purchasing, Requisition Entry Defaults).

    • Right-click in the window and select Add Row.

    • Select the new Group ID, a Track ID, and any of the check boxes as necessary.  Click F1 for more information about the Requisition Entry Defaults window.  If you would like to create a new approval track to use, refer to the Setup of Approval Tracks topic (F1 Help, Purchasing Table of Contents, Approval Tracks, Setup of Approval Tracks).

Step 5:  Set Role Security Permissions for Each Portlet

From each portlet, click the wrench icon to access customizable features.  Use the Security tab to assign operations to roles by selecting the corresponding check box where applicable.  Below are the portlets for which role security permission must be set.

  • Account Notification

    • From the Account Notification portlet, click the wrench icon to open the Portlet Settings and Preferences screen.

    • Click the Security tab.

    • Select the appropriate Operations check boxes for each role or select the All Operations check box.

    Cancel Notification

    Selecting this check box for a role allows persons with that role to cancel an account notification for a JICS user via the Held Notifications tab on the View New Accounts screen.

    Send Email

    Selecting this check box for a role allows persons with that role to send an email with login and password information to a JICS user via the View New Accounts screen.

    Can Set Up Portlet

    Selecting this check box for a role allows persons with that role to see the Setup link in the Account Notification toolbar.

    Create Email Template

    Selecting this check box for a role allows persons with that role to create a new email template to use when sending email with login and password information to a JICS user via the View New Accounts screen.  When this permission in given, the add template button is available on the Setup screen.

    Modify Email Template

    Selecting this check box for a role allows persons with that role to edit an existing email template.  This template is used when sending email with login and password information to a JICS user via the View New Accounts screen.  When this permission is given, the edit template button is available on the Setup screen.

    Move to Active

    Selecting this check box for a role allows persons with that role to move an account notification from Held to Active status on the View New Accounts screen.

    Move to Held

    Selecting this check box for a role allows persons with that role to move an account notification from Active to Held status on the View New Accounts screen.

    • Save.

    • If you need to add a role, click the Add a Role link.  See Getting Started with JICS for more information about adding a role.

  • Employee Information

    • From the Employee Information portlet, click the wrench icon to open the Portlet Settings and Preferences screen.

    • Click the Security tab.

    • Select the appropriate Operations check boxes for each role or select the All Operations check box.

    Note: This portlet does not have settings and preferences.

    • If you need to add a role, click the Add a Role link.  See Getting Started with JICS for more information about adding a role.

     

  • GL Account Lookup

    • From the GL Account Lookup portlet, click the wrench icon to open the Portlet Settings and Preferences screen.

    • Click the Security tab.

    • Select the appropriate Operations check boxes for each role or select the All Operations check box.

    • Note: This portlet does not have settings and preferences.

    • If you need to add a role, click the Add a Role link.  See Getting Started with JICS for more information about adding a role.

     

  • Item Approval

    • From the Item Approval portlet, click the wrench icon to open the Portlet Settings and Preferences screen.

    • Click the Security tab.

    • Select the appropriate Operations check boxes for each role or select the All Operations check box.

    • Note: This portlet does not have settings and preferences.

    • If you need to add a role, click the Add a Role link.  See Getting Started with JICS for more information about adding a role.

     

  • Requisition Entry

    • From the Requisition Entry portlet, click the wrench icon to open the Portlet Settings and Preferences screen.

    • Click the Security tab.

    • Select the appropriate Operations check boxes for each role or select the All Operations check box.

    Can Check Requisitions

    Selecting this check box for a role allows persons with that role to access the Check Requisition Status link on the Requisition Entry portlet.  Follow this link to conduct a search on requisitions in the CX database.  

    Can Enter Requisitions

    Selecting this check box for a role allows persons with that role to access the Enter Requisitions link on the Requisition Entry portlet.  Follow this link to create a purchase order request for one or more items.  The data you enter is stored in the CX database when you save the requisition.

    • Save.

    • If you need to add a role, click the Add a Role link.  See Getting Started with JICS for more information about adding a role.

     

  • Timecard

    • Security is set by the JICS administrator on the Global Portlet Operations screen.  See Getting Started with JICS for more information on setting global permissions.

Step 6:  Assign Staff Administrators Responsible for Account Notification

  • Request the JICS Administrator to assign the Staff Administrator who will be responsible for sending account notifications to Staff and other Staff Administrators.

Step 7:  Configure the Account Notification Portlet

  • From the Account Notification portlet, click the Site Manager link in the upper right corner of the screen.

  • Click the Edit link for Global Portlet Operations.

  • Scroll down to locate Global Operations for Account Notification.

  • Select the appropriate roles for each of the following operation:

    • Can Receive Account Notification?

      Users in the selected groups will receive account notifications.  The following link may display on the Account Notification portlet when they log in to JICS:  View Active and Held Account Notifications to Send.

  • Save.

  • Exit.

Only ONE person, who has been assigned the account notification responsibility, will need to configure the Account Notification portlet.

  • Click the Setup is required before account notification will perform properly link on the Main screen, which opens the Setup screen.  Enter the following configuration information:

    • From the Security Groups to Send Account Notifications to drop-down list, select the appropriate security group to which to send account notifications.  Security groups in this list have been given the Can Receive Account Notification global operation and have not already been assigned to another instance of the portlet.  For more information, see the Account Notification topic in Getting Started with JICS.

    • Select the Default Email Template for Account Notifications.  Displayed are the email templates available for the security group selected above in the Security Groups to Send Account Notifications to list.

You can also create new or edit existing email templates that will be used by the Account Notification portlet.  These templates should include the cc, bcc, subject, and body of the email and can be customized to include the user's name, major, login, and password.  The options to create or modify email templates are available if Modify Email Template and Create Email Template check boxes were selected as operations for the security group.  For more information, see the Email Templates topic in Getting Started with JICS.

    • In Account Notification Delivery Method, decide if staff will be notified by automatic email or manual email that an account has been created for them.

If Automatic Email Notification is selected, an email will be automatically sent without any intervention required by the Staff Administrator.  This option is not recommended for the initial batch account notification.

If Manual Email Notification is selected, a temp table in JICS will collect the new accounts and the Staff Administrator will review and send emails through the Account Notification portlet.  This option is recommended for the initial batch account notification.

Note: For more information on the Account Notification portlet, see Getting Started for JICS.

Step 8:  Configure the Timecard Portlet

Your role security (see Getting Started with JICS for more information) gives you the ability to define pay periods, create pay period/group combinations, select the current pay period to use for timecard recording, and upload the timecards to your ERP system for processing.  Once the pay period/group combinations are defined, you and your fellow employees can use the Timecard portlet to report your hours worked and time off benefits used.  After the timecard information is reported, you can upload the timecards to the EX system.

  • Log into JICS and open the Timecard portlet.

  • Click the Pay Period Administration link to create pay period/group combinations and make a pay period active by selecting it as the default.

    • To create a new pay period/group combination - click add, enter a description to identify the pay period, use the calendar to select the Pay Period Begin, End, and Check Dates, select a Pay Group, click Save.  

Note 1: The Pay Group drop-down menu is populated using values from the EX database.

Note 2: Make sure the Date values you enter in this portlet are equal to those stored on the Payroll Controls window in EX (EX, Human Resources module, Payroll, Admin, Task, Configuration, Payroll Configuration, Payroll Controls).  For more information about this window, click F1 to access the Payroll Control window Help topic.

    • Use the appropriate check box to mark the default pay period and click Save.  Now when employees click the Enter Time link, they will be able to record their payroll information for the default pay period.

  • Upload the Timecard information for a pay period/group combination to EX for Payroll Processing.  For more information about Payroll Processing in EX, refer to the F1 Online Help topic titled Running a Regular Payroll.

    • Identify the Timecards to upload by selecting the appropriate Pay Group from the drop-down menu.  

    • Update the Payroll information in the EX database by clicking the Update Timecards button, which uploads the Timecard information for the Pay Group selected.

Step 9:  Provide Login and Password Information to Existing Staff

Once Staff members are assigned to the Staff role within EX, and the DIS sends this information to JICS, the Account Notification portlet will allow Login and Password information to be distributed via email.

PX Functionality

6.4 Release

No changes or enhancements were made to PX to accommodate Staff functionality.

6.0 Release

Some changes and enhancements were made to the PX products to accommodate Staff.

Fiscal Reporting V6.2 System Enhancement for Lookup

  • To allow quick lookup access to the un-posted transactions, the GLACCT field has been added as a secondary key to the FISJRNLS, FISPAYDET and INTERFACE data files.  These keys allow duplicates.

  • To limit users to specific accounts for inquire and the requisition entry use the command procedures for the security option of FIS_STATUS.  When using ADD_USERNAME, enter the JICS ID instead of the VMS username.  Modifications to STATUS_SECURITY to specify the GENLED accounts for the pointer file will remain the same.  The JICS ID will be the person's ID in dbCENTREL.

  • GL_SEC_PTR logical is defined to identify the location of the security points and GL_SECURITY$ must be defined as "YES"

Payroll V5.2 Systems Enhancements for Timecard

  • To provide a unique 'Pay Group' name for the timecard administration, a new field is added to the PAYCODE file.  Data will be entered in the new field for the group code, GRP, records.

  • The new field, ICPGRPNAME, is a 10 character alphanumeric field.  The description is "Group name for ICP."

  • Add this field to the SMPL screen format to be prompted.

  • Update all GRP records and enter a name into the new field.

  • For the time card entry, the position code used in Personnel's PRSPOSITION file was added to the wage records of PAYREG.  The description of the position code is displayed as the "Position Title".  This is also used to display position information.

  • Add the POSITION field to the PAYREG FULL, HRLY, and SALD screen formats to be prompted.  The field should validate against the PRSFIL:PRSPOSITION file.

  • Update all active wage records with a position name.

  • The timecard information is stored in a new file, PAYICPTC, which is located in the Payroll System file directory.

Personnel V1.1 System Enhancements for Timecard

  • Adding and updating dependents directly updates PRSDEPENDENT.  However, the data related to DBCMAIN (phone, work phone, physical restriction, and address) is stored in the PRSICPUPD file for approval.

  • The PRSICPUPD data file must be located in the Personnel System file directory.

Distributed Purchasing System (DPS) Enhancements for Requisitions and Item Approval

  • To provide a link between the authorization and proxy access defined in DPS_AUTHORIZE and the JICS host Ids, a new field must be added to the DPS_AUTHORIZE file.   

  • The new field, JICSID, is 9 character alphanumeric field. The description is "JICS ID".

  • Add the JICSID field as a secondary key that allows duplicates.

  • Add this field to the AUTHORIZE screen format to be prompted.

  • Update the authorization records and associate the JICS id with the usernames. Only authorization records that have a JICS ID will be able to process requisitions and approvals.

  • To allow quick access to items for approval, the ALPHA_USER field in DPS_AUTHORIZE is added as a virtual field in the REQV2, DPS_AUDIT and REQ_MASTER files.

  • Add a virtual field to REQV2. The local field to key on is BATCHOWNER using the primary key. And the foreign field is ALPHA_USER.

  • Add a virtual field to DPS_AUDIT. The local field to key on is USERNAME using the primary key. And the foreign field is ALPHA_USER.

  • Add a virtual field to DPS_AUDIT called APPROV_NAME. The local field to key on is APPROV_USER using the primary key. The foreign field is ALPHA_USER. The explanation is “Approving User’s Name” and the Heading and Prompt are “Approver”.

  • Add a virtual field to REQ_MASTER. The local field to key on is DOC_OWNER using the primary key. The foreign field is ALPHA_USER.

  • If the client would like to provide a catalog lookup the CAT_FILE and CAT_FIELDS logicals must be defined. CAT_FILE defines the data file where catalog data will be stored. A suggested file, DPS_CATALOG, is provided.  CAT_FIELDS lists the data file fields to be used for the lookup. The order of the field names is important and as follows: part number, description, unit price, catalog number (optional), and unit type (box, dozen, gallon, etc).

  • The new field, JICS_FLAG is a one-position alphanumeric field. The description is “JICS Flag”.

  • Update the DPS_PROXY records and enter a Y into the JICS_FLAG field to enable proxies for that user.

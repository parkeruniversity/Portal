Setup Steps for EX

In order to successfully implement CRM Student, the following steps should be performed in the order in which they are listed.

NOTE:  Your JICS Administrator has already defined roles that have access to individual portlets via the Allow Access check box (Site Manager link, Global Portlet Operations Edit link).  For more information about the Allow Access global operation, see the Getting Started with JICS.

Step 1: Create Advisor and Student Administrator JICS accounts in Jenzabar EX

  • Select the individual who have Student Administrator privileges and add that role to their ID Number.

    • This is done in the Web Administration module.  For more information, see the Group Membership section of the EX Functionality topic.

  • Select the individual who have Advisor Administrator privileges and add that role to their ID Number.

    • This is done in the Web Administration module.  For more information, see the Group Membership section of the EX Functionality topic.

  • Retrieve from EX the login and password for this individual and provide them with this information.

Step 2: Activate CRM Student in Jenzabar EX

Certain elements of CRM Student need to be activated from within Jenzabar EX to function properly.  To activate these elements:

  • Log into Jenzabar EX as the administrator

  • Open the Maintain Config Table window (Common, Task, Configurations, Maintain Configurations).

  • To ensure that JICS is enabled, scroll down to JICS (Module), BASE (Function), ENABLED (Characteristic) and set the Configuration Value to 'Y'.  To disable JICS, set the Configuration Value to 'N'.

  • To ensure that CRM Student is enabled, scroll down to JICS (Module), STUD (Function), ENABLED (Characteristic) and set the Configuration Value to 'Y'.  To disable CRM Student, set the Configuration Value to 'N'.

  • To display Cross-Listed Courses in JICS, scroll down to RE (Module), XLISTED_CRS (Function), LISTS (Characteristic) and set the Configuration Value to 'COMBINED_LISTS' to display cross-listed courses.

 

Step 3: Configure the Advisor Meetings portlet

  • In JICS:

    • Log in to JICS as the JICS Administrator and click on the Site Manager link.

On the Site Settings tab of the Site Manager screen, click the Edit link or icon () next to the Global Portlet Operations option to display the list of portlets.  

Click the Advisor Meetings link to access its Global Portlet Operations.

In the Can the user change the global settings column, select the checkbox for each Role that will be able to access the Setup menu.

In the Is Advisor column, select the checkbox for each Role that will operate in the capacity of an Advisor (only these users will be able to set up meetings with Advisees).

    • Click the Setup menu link from the Advisor Meetings portlet and set the following option:

Select the Show Notes to Students check box if students will be allowed to view the notes from advisor meetings. If it is unchecked, the notes will not be displayed to students.  This checkbox will not affect advisors: they will always be able to view notes on a student.

Click Save to save your changes and return to the Advisor Meetings screen.

Please note that only those logged in under a role that was given permission to change Global Settings will have the Setup button available.

Step 4: Configure the Course Needs portlet

  • In Jenzabar EX:

    • Several portlets, including Grade Entry, Unofficial Transcript, Student Schedule and My Account Info allow the user to create custom reports that the user can see from within the portlet.  During installation, several reports (InfoMaker .pbl files) are copied into default directories on the web server that these portlets access.  If these reports are not in the default location, you will need to use the Web Administration module to point JICS to these reports properly.

    • To update the location of the customized reports for Course Needs:

Open the Web Administration module and select Task > Web Administration > Configuration > UI Configuration.

In the Category Name drop-down box, select Advising

Type the location of the .pbl for the module in the Value column for ReportPBLPath.

Type the name of the report in the .pbl you entered above in the Value column for AdvisingWorksheet.  This affects the report used in the Course Needs portlet.

  • In JICS:

    • Log in to JICS as the JICS Administrator and click on the Site Manager link.

On the Site Settings tab of the Site Manager screen, click the Edit link or icon () next to the Global Portlet Operations option to display the list of portlets.

Click the Course Needs link to access its Global Portlet Operations.

In the Can the user change the global settings column, select the checkbox for each Role that will be able to access the Setup menu.

    • Click the Setup menu link from the Course Needs portlet and set the following options:

Select a report to be made available for printing (if any) from the Reports to View drop-down box.

Enter the name of the Custom Report Link in the Custom report Link Text text box.  If the Reports to View drop-down box is set to None, the information in this text box will not be used by the portlet.

Select the Allow Recalculate Student Progress check box ifadvisor and student users of Course Needs or the Major Exploration and What If Scenarios portlets will be allowed to recalculate student progress, applying any recently completed or updated course work.

Click Save to save your changes and return to the Course Needs screen.

Please note that only those logged in under a role that was given permission to change Global Settings will have the Setup button available.

Step 5: Configure the Course Schedules portlet

  • In Jenzabar EX:

    • Open the Web Registration Group Definition window (Web Administration, Task, Web Administration, Configuration, Web Registration Group Definition)

For each Group Code, select the appropriate Web Registration Requirement (N - None Required, A - Advisor Approval Required or C - Advisor Clearance Required).  Note that Advisor Clearance Required puts a hold on the student which prevents them from registering until their advisor has cleared them to register.

    • If links to your online catalog are to be used, open the Catalog Master Extension window (Web Administration, Task, Web Administration, Configuration, Catalog Master Extension) and enter the Catalog URL into the text box provided for each course.

    • Open the CRM/Web tab in the Courses window (Registration, Task, Course, Courses).  For each Course, select the following as appropriate:

On the Controls sub-tab:

In the Show On Web drop-down menu, select how the class will be displayed on the web.  You may choose to have the class only show on the Web (in CRM Student) or only in LMS as well as displaying the class in both locations (Both) or neither (None).

In the CRM Display of Crosslisted Courses drop-down menu, select how crosslisted courses will be displayed.  You may choose to Display Child Course(s) on Parent Course List, Display Child Course(s) and Parent Courses Separately or choose not to display crosslisted courses in conjunction with one another (None).

Select the Must Pay Course checkbox if the course must be paid for before registration in the course can be completed.  If left unchecked, registration for the course can be completed without payment.

Select the Available as a Public Registration Course checkbox if the course is to be available for guest registration.  If left unchecked, the course will not appear for guest logins.

Select the Requires Course Authorization for Registration if Advisor Clearance is being used.  If left unchecked, the user can register for a course without receiving Clearance from his Advisor.

Enter the date range users will be permitted to add courses in the Add Begin Date and Add End Date text boxes.

Enter the date range users will be permitted to drop courses in the Drop Begin Date and Drop End Date text boxes.

Enter the date range faculty will be permitted to enter final grades in the First Final Grades Date and Last Final Grades Date text boxes.

Enter the date range faculty will be permitted to enter midterm grades in the First Midterm Grades Date and Last Midterm Grades Date text boxes.

On the Restrict Group/Course Registration subtab:

Select Course from the Type of Registration Limit drop-down list box if students have to be in a selected Web Registration Group to register for the course.  Select Group if students can only register for the course if they are in a selected Web Group.

Select a group from the Restrict Course Registration by Web Group column on the left and click Add to add it to the Selected Web Registration Groups column on the right.  To remove a selected group, select that group from the Selected Web Registration Groups column on the right and click the Remove button.  Only selected groups will be allowed to register for this course.  If no groups are selected anyone may register for this course.

Open the Configuration Table tab on the Registration Configuration window (Registration Admin, Task, Configuration, Registration Configuration) and select the following:

Select the Use automatic Drop for expired Reserved Courses checkbox if you wish to drop Must Pay or Advisor Clearance classes automatically after a certain period of time.  Leave this checkbox empty if you do not wish to drop courses automatically.  If you do choose to use this option, enter how often, in minutes, the system should check for expirations in the text box provided and select the following, as appropriate:

Select the For Courses Requiring Payment checkbox if Must Pay courses will be dropped automatically by the system.  Leave this checkbox empty if you do not wish to automatically drop Must Pay courses.  If you do choose this option, enter how long long, in hours, a Must Pay course may remain in reserved status in the text box provided.

Select the For Courses Requiring Advisor Approval checkbox if Advisor Clearance courses will be dropped automatically by the system.  Leave this checkbox empty if you do not wish to automatically drop Advisor Clearance courses.  If you do choose this option, enter how long, in hours, an Advisor Clearance course may remain in reserved status in the text box provided.

Select the E-mail Student of impending Drop of Reserved Course? checkbox if you want the system to automatically e-mail a warning to the student before the course is automatically dropped.  Leave this checkbox empty if you do not want an e-mail warning to be sent to the student.  If you do choose this option, enter how long before the course is dropped you want the email to be sent, in hours, in the text box provided.

You can choose how you want CrossListed courses to display in the Display of Crosslisted Courses drop-down list box.  You can select Display Child Course(s) on Parent Course List or Display Child Course(s) and Parent Courses Separately.

Check the Use Course Authorization checkbox if Advisor Clearance will be used to put an added course in Reserved status pending approval of an Advisor.

  • In JICS:

    • Log in to JICS as the JICS Administrator and click on the Site Manager link.

On the Site Settings tab of the Site Manager screen, click the Edit link or icon () next to the Global Portlet Operations option to display the list of portlets.  

Click the Course Schedules link to access its Global Portlet Operations.

In the Add/Drop Courses for Others column, select the checkbox for each Role that can Add/Drop Courses for students.

In the Add/Drop Courses for Self column, select the checkbox for each Role that can Add/Drop Courses for themselves.

In the Can Admin Global Settings column, select the checkbox for each Role that will be able to access the Setup menu.

    • Open the CRM Module Permissions tab on the Permissions window (Registration Admin, Task, Configuration, Permissions).  For each Group in the left-hand column, select the following:

Select the Override Advisor Registration Clearance Errors if users in that group will be able to override Advisor Clearance.  Leave the box unchecked if the users will not be able to override errors.

Select the Override Change Advisor Approval Required Course to Current Errors if users in that group will be able to override the Advisor Approval Required error.  Leave the box unchecked if the users will not be able to override errors.

Select the Override Change Must Pay Course to Current Errors if users in that group will be able to override a Must Pay error.  Leave the box unchecked if the users will not be able to override errors.

Select the Override Course Authorization Required for Registration Errors if users in that group will be able to override a Course Authorization error.  Leave the box unchecked if the users will not be able to override errors.

Select the Override Web Group Registration Limits of "Course" Errors if users in that group will be able to override Registration Limits by Course error.  Leave the box unchecked if the users will not be able to override errors.

Select the Override Web Group Registration Limits of "Group" Errors if users in that group will be able to override Registration Limits by Group error.  Leave the box unchecked if the users will not be able to override errors.

    • Click the Setup menu link from the Course Schedules portlet and set the following options:

Select the Form you wish new users to use when registering for a login in the Public Registration Login Password Request Form drop-down list box.

Select the Display All Courses, including Courses the Student Is Not Allowed to Add check box if users will be allowed to view all courses. If it is unchecked, only courses the student is allowed to add be displayed.

Select the On the Details Screen, Display the Link to a Catalog Page for each Course check box if users will have a link to the catalog page for each course on the Details screen. If it is unchecked, the link to the catalog page will not be displayed.

Select the On the Course Textbooks Screen, Display Textbook Prices check box if users will be able to view the price of a textbook on the Course Textbooks screen. If it is unchecked, textbook prices will not be displayed.

Select the Use the Advisor Approval Process for Selected Students check box if the Advisor Approval Process is to be used. If it is unchecked, the system will not check to determine if Advisor Approval is needed.

Select the Use the Registration Clearance Process for Selected Students check box if the Registration Clearance  Process is to be used. If it is unchecked, the system will not check to determine if clearance is needed.

Select the Use the Must Pay Course Process for Selected Courses check box if the Must Pay Course Process is to be used. If it is unchecked, the system will not check to determine if a course is a Must Pay course.

Select the Allow Guests to Request a Login so they can Register for Courses not Requiring Matriculation check box if guests will be able to request a login using the form specified at the top of this screen. If it is unchecked, the system will not permit guests to request a login automatically.

Select the Allow Students to Edit the Credits Column for Courses with Variable Credit check box if the students will be permitted to enter their own credits for variable credit courses.  If it is unchecked, students will not be able to enter their own credits.

Select the Allow Students to Add Themselves to a Waitlist for a Course that is full check box if students will be permitted to add themselves to a waitlist. If it is unchecked, students will not be able to add courses that are full.

Step 7: Configure the Financial Aid Administration portlet

Note: This portlet will not function if your organization is not licensed for PowerFAIDS.

  • In Jenzabar EX:

    • On the Financial Aid Interface Configuration screen (Financial Aid > Task > Administration > Financial Aid Interface Configuration) perform the following:

Click the PowerFAIDS Database Connection button.

Choose the appropriate ODBC connection in the dropdown provided.

If you are on another machine, you must first create an ODBC connection with the exact same name on that machine so that it appears in the dropdown.

  • In JICS:

    • Click the wrench () from the Financial Aid Administration portlet and set the following options:

On the Settings tab:

Enter the subject of the alert in the Missing Document Alert Message Subject text box.  This will be the subject of the email sent to a student to notify them of a missing financial aid document.  The alerts functionality is only available if the Financial Aid Document Tracking portlet is accessed through the Financial Aid Administration portlet and the Can Send Alerts permission is granted to the logged-in user.

Enter the text of the alert in the Missing Document Alert Message Template text box.   This will be in the message body of the email sent to the student to notify them of a missing financial aid document.  The document name and award year may be dynamically included in the body of the alert if the @@Doc and/or the @@YEAR variables are specified in the template text.  The alerts functionality is only available if the Financial Aid Document Tracking portlet is accessed through the Financial Aid Administration portlet and the Can Send Alerts permission is granted to the logged-in user.

Click the Save button.

On the Security tab:

Select the check boxes beside the particular roles you want to be able to send alerts for missing Financial Aid documents in the Can Send Alerts column.  This causes the Alerts check box to be displayed next to each item listed in the Missing Documents section of the Financial Aid Document Tracking portlet for users of the selected roles.  The alerts functionality is only available if the Financial Aid Document Tracking portlet is accessed through the Financial Aid Administration portlet and the Can Send Alerts permission is granted to the logged-in user.

Click the Save button.

Step 17: Configure the Student Schedule portlet

  • In Jenzabar EX:

    • Several portlets, including Grade Entry, Unofficial Transcript, Student Schedule and My Account Info allow the user to create custom reports that the user can see from within the portlet.  During installation, several reports (InfoMaker .pbl files) are copied into default directories on the web server that these portlets access.  If these reports are not in the default location, you will need to use the Web Administration module to point JICS to these reports properly.

    • To update the location of the customized reports in Student Schedule:

Open the Web Administration module and select Task > Web Administration > Configuration > UI Configuration.

In the Category Name drop-down box, select Registration.

Type the location of the .pbl for the module in the Value column for ReportPBLPath.

Type the name of the report in the .pbl you entered above in the Value column for StudentSchedule.  This affects the report used in the Student Schedule portlet.

  • In JICS:

    • Log in to JICS as the JICS Administrator and click on the Site Manager link.

On the Site Settings tab of the Site Manager screen, click the Edit link or icon () next to the Global Portlet Operations option to display the list of portlets.

Click the Student Schedule link to access its Global Portlet Operations.

In the Can Admin Global Settings column, select the checkbox for each Role that will be able to access the Setup menu.

    • Click the Setup menu link from the Student Schedule portlet and set the following options:

Select the Textbook Display check box if the system will display the Textbook icon on the Course Schedules Results screen and display Textbooks on the Student Schedule if textbooks are associated with the course.  If left unchecked, textbook information will not appear.

Select whether a Custom report or no report will be made available for printing (if any) from the View Custom Report drop-down box.

Enter the name of the Custom Report Link in the Custom Report Link Text text box.  If the View Custom Report drop-down box is set to None, the information in this text box will not be used by the portlet.

Click Save to save your changes and return to the Student Schedule screen.

Please note that only those logged in under a role that was given permission to change Global Settings will have the Setup button available.

Step 18: Configure the Student Survey portlet

  • In Jenzabar EX:

    • Open the Student Life Configuration window (Student Life, Task, Module Wide Configuration, Student Life Configuration) and select the Residence Life tab.

    • If you wish to use JICS to conduct housing surveys, make sure the Use JICS Housing Surveys checkbox is checked.  If you do not wish to use JICS for housing surveys, leave this unchecked.

    • If you are going to use JICS to conduct surveys, you must enter address codes in the fields provided.  Note: If you have configured EX for the Residence Info portlet, you should have already entered this information.

Enter the email code you wish the survey to use for the Value of What addr code is used for email for students in Res Life?  In a default setup, this would be *EML.

Enter the address code you wish the survey to use for the Value of What addr code is used for students in Res Life?  In a default setup, this would be *LHP.

    • If you are going to use JICS to conduct surveys, you must create a survey code in EX:

Open the Maintain Surveys window (Student Life, Task, Residence Life, Surveys, Maintain Surveys)

Select the year and term in the Session drop-down list box.

Right click on the list of surveys at the bottom and select Add Row.

Enter the Survey code, Description and Cutoff Priority for the survey.

The Session and Survey code will be used when you generate the survey.  Please see Form Administration for more information on building forms.

  • In JICS:

    • Click the Setup menu link from the Student Survey portlet to set up your forms.  See Form Administration for more information on how to create and manage forms.  Please note that only those logged in under a role that was given permission to change Global Settings in Step 3 will have the Setup button available.

    • Click the wrench () from the Student Survey portlet and set the following options:

On the Settings tab:

Use the Form Type drop-down list box to select the form to be used in this portlet.  In most cases, this will be the StudentSurvey form type.  

Click Save to store this information.

On the Security tab:

Assign Permissions for each Role to limit who can use this portlet to create a form, delete a form, modify a form and set up the portlet.

Click Save to store this information.

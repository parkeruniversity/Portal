Course Authorization

The Course Authorization portlet is available only if you have permission to do course authorizations.

The Course Authorization portlet allows persons with Faculty and/or Advisor roles to pre-approve an override to a registration error.  You can select a course and students for whom to provide authorization to register for the course even if the student has not met a course requisite, the course is full, or the student has a schedule conflict.  If you have the proper permissions, the Condition Type errors for which you can provide authorization are Capacity, Requisite, and Section Conflict.  You can also indicate your preference of criteria when doing a Name Search.

The default screen displays the following:

  • Explanatory text; if authorizations have not been enabled and you do not have permissions to view course authorizations, the text will include this message:  "Course Authorizations are not enabled.  You will not be able to access the portlet functionality."

  • Current term+

  • Division

  • Your name

  • List of the courses you are teaching for the current term; each course is a link to the Course Authorization Details screen

  • Catalog description (CX only)

  • Total number of authorizations that have been granted for each course

  • Course List link; allows you to access the Course Authorization Course List screen where you can select any course for which you have authorizations to view and/or maintain if you have permission to access courses other than the ones you are teaching

 

Set Options

The Set Options link allows you to change your personal options and indicate the options you would like to use when doing a Name Search.  You can also indicate which column in the Authorizations Granted table you would like to use for sorting when the Course Authorization Details screen opens.

  • Name Search Options

    From the Course Authorizations portlet:

    • Click the Set Options link.

    • Under the Name Search Options, select one of three options to use when doing a Name Search:

      • Select the Search Using Phonetic Match check box if you want Name Search to attempt to find a name using a phonetic or Soundex match.

      • Select the Search Using the Primary Name check box if you want Name Search to attempt to find a name using the LAST_NAME, FIRST_NAME, and/or MIDDLENAME columns in the ID record.

      • Select the Search using the Alternate Name Type check box if you want Name Search to attempt to find a name using names other than the main name record.  Other name types could include birth name, nickname, preferred name, former name, etc.

    • Click the Save button.

  • Default Authorizations Sort Order

    From the Course Authorizations portlet:

    • Click the Set Options link.

    • Under Default Authorizations Sort Order, the Sort Column drop-down lists three columns:  Student, Condition, and Reason.

    • Select the column on which you want the Authorizations Granted table to be sorted when the Course Authorization Details screen opens or when you select a different course.

    • Click the Save button.

 

Setup

  • Allow Users to Add and Delete Requisite Authorizations

    From the Course Authorizations portlet:

    • Click the Setup link.

    • Select this check box if you want to allow faculty or other staff to authorize that selected students can register for a course in spite of the requisite error at the time registration occurs.  

      If the Use Course Authorization Functionality in CRM Faculty is selected and the Allow Requisite Authorization check box is selected, then Requisites will appear as an option in the Conditions drop-down if the faculty/staff has rights to maintain requisites and the Requisites count column will appear on the Course Authorization and Add New Authorizations screens.

  • Allow Users to Add and Delete Capacity Authorizations

    From the Course Authorizations portlet:

    • Click the Setup link.

    • Select this check box if you want to allow faculty or other staff to authorize that selected students can register for a course in spite of the Course Capacity error at the time registration occurs.

      If the Use Course Authorization Functionality in CRM Faculty is selected and the Allow Capacity Authorization check box is selected, then Capacity will appear as an option in the Conditions drop-down if the faculty/staff has rights to maintain capacity and the Capacity count column will appear on the Course Authorization and Add New Authorizations screens.

  • Allow Users to Add and Delete Schedule Conflict Authorizations

    From the Course Authorizations portlet:

    • Click the Setup link.

    • Select this check box if you want to allow faculty or other staff to authorize that selected students can register for a course in spite of the schedule conflict error at the time registration occurs.

      If the Use Course Authorization Functionality in CRM Faculty is selected and the Allow Schedule Conflict Authorization check box is selected, then Schedule Conflicts will appear as an option in the Conditions drop-down if the faculty/staff has rights to maintain schedule conflicts and the Schedule Conflict count column will appear on the Course Authorization and Add New Authorizations screens.

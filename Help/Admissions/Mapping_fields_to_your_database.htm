Mapping fields to your database

Candidate allows you to configure forms so that data entered by users is automatically imported into your database. In order to take advantage of this functionality, you must map fields in your form to specific tables.

To map fields to your database:

  1. If the form is not already open for editing, open it, as described in Opening a form for editing. Note that this page includes a column labeled Db Mapping, which shows whether each question has been mapped to the database.

  2. If the field has not been mapped, click the link labeled Not mapped. The system displays a screen with the label “Mapping Step One: Choose a Table.”

  3. Select the table to which you want to map this question. The system displays a screen with the label “Mapping Step Two: Set up mapping.” If you have already mapped other questions to this table, the exact mappings are displayed

  4. Use the drop-down lists to match fields in the table with data that will be derived from the form. Note that you have the option to add a row to the table if necessary.

Payment Administration

If appropriate, you can use the Payment Administration portlet to set up your portal to allow payments to be made through certain portlets (which typically are available only if you have purchased certain CRMs). When you do this, these portlets direct the user to a Payments portlet at certain points. For example, this might be happen in any of the following situations:

·       When a constituent needs to submit payment for a gift or pledge.

·       When a candidate needs to pay an application fee.

·       When a student needs to make a payment.  

Additional configuration settings within a specific CRM may be required to allow the following functionality:

·       Allow the user to pay by credit card or electronic check, which creates financial transactions in the ERP system.

·       Allow the portal to interact with Authorize.net to verify payments and update financial transactions in the ERP system. Authorize.net is a third-party software provider that interfaces with JICS.

+ To give a role permission to access the Payment Administration portlet

1.        Click the wrench icon from the Payment Administration portlet.

2.        Click the Permissions tab.

3.        For any role that should access to the Payment Administration portlet, select the Can Administer Payments checkbox.

4.        Click Save.

 

Managing payments

To manage payments, navigate to the Payment Administration portlet and click the link labeled Click here to administer the Payments Portlet link. The portlet will display a form.

Each of the form's options is explained more below.

 

Work with Credit Card

Clicking this link displays the Credit Card Configuration screen. On this screen, you can add, modify, or remove credit cards, as follows:

+ Add a credit card to the system

The Add a Credit Card screen allows you to add credit cards available for processing by the Payments portlet.

  • From the Credit Card Configuration screen, click the Add a credit card to the system link.

  • Enter the name of the credit card you want to add.

  • Enter the description of the credit card you want to add.

  • Click the Add button.

  • If the addition of the credit card was successful, the Credit Card Configuration screen reappears and displays the following message:  "The insert of the credit card was successful."

+ Modify an existing credit card's properties

The Select Credit Card screen allows you to select and modify credit cards available for processing by the Payments portlet.

  • From the Credit Card Configuration screen, click the Modify and existing credit card's properties link.

  • Select the credit card you want to modify.

  • Enter any modifications you want to make in the Credit Card and Credit Card Descriptions fields.

  • Click the OK button.

  • If the modification of the credit card was successful, the Credit Card Configuration screen reappears and displays the following message:  "The modification of the credit card was successful."

+ Remove a credit card from the system

The Select Credit Card screen allows you to select and remove credit cards available for processing by the Payments portlet.

  • From the Credit Card Configuration screen, click the Remove a credit card from the system link.

  • Select the credit card you want to remove.

  • Click the OK button.

  • If the removal of the credit card was successful, the Credit Card Configuration screen reappears and displays the following message:  "The removal of the credit card was successful."

+ Exit credit card administration

This link returns you to the Payment Administration screen.

Additional options

You also have the following options:

+ Is the Payment Portlet in testing mode?

  • In order to verify its functionality, we recommend that you select the testing mode (select Yes radio button) prior to making this portlet available to your entire Internet campus community.  However, it is important to note that Authorize.net does not allow electronic checks to be tested.

  • After you have tested and verified the payment process, you can select the No radio button to deactivate the testing mode and confirm your Authorize.net required data.

+ What is your Authorize.net login ID?

  • This Login ID is provided by Authorize.net to establish the working relationship between your organization and Authorize.net

+ What is your Authorize.net Transkey?

  • This value is provided by Authorize.net to ensure your organization's proper setup.

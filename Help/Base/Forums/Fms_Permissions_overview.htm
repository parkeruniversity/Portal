Permissions overview

Most permissions in the Forums portlet are specific to a single instance of the portlet. For this reason, you might have broad permissions in one portlet instance but not another.

Some tasks you might be allowed to do include the following:

·       Publishing posts.

·       Publishing replies to posts.

·       Editing and deleting your own posts and replies.

In general, you can tell whether you have permission to do something if the appropriate buttons are displayed. For example, if you see the Add a Post button, you have permission to post. However, there are some cases where you have permission to post but the button does not display -- for details, see "Why can't I post?" below.

If you are unable to do something you think you should have permission to, consult an administrator for your portal.

+ Why can't I post?

Regardless of your permissions, your ability to post and reply might be restricted in the following situations:

·       Some posts may be set up to allow only one reply per user -- so if you have already responded once, the Reply button will not display again.

·       If a topic has been set up to be read-only, the Add a Post and Reply buttons will not display in that topic, regardless of what permissions you have. You can tell whether a topic is read-only if, when you view it from the category level, it is marked with an eye icon.

+ Additional options for administrators, faculty

If you are an administrator or a faculty member, on many screens described in these help pages, you will have additional options beyond what is described here. For more details on these, or for help managing a Forums portlet instance, see Jenzabar's Internet Campus Solution: Administration Guide or e-Racer: Faculty Guide.

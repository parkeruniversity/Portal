Portlet Specific Payment Configuration

This screen opens when you use the option labeled Work with portlet specific payment configurations in the Payment Administration portlet. You can enter the following specific configuration settings for each type of payment.

Configure Payment Information

+ Payment Config Description

Enter a description of this payment configuration.

+ Transaction code

This value is defined by the ERP system used to record fees in Business Office and enables communication between the Payments portlet and the ERP. It is generally an ERP-specific code that may be needed in order to record the payment in the proper general ledger accounts.

For EX, this is a receipt code that is used to determine the debit and credit accounts to use to record a payment transaction. The receipt codes are defined on the Receipt Code Control window in the Accounts Receivable Administration module.

For CX, general ledger account numbers would be entered directly in the Transaction Code column.

For PX, please refer to the PX Admissions System Reference Manual, "Appendix D, PX Admissions System and JICS."

+ Authorize payments

This radio button indicates whether the payment is actually charged to the person's credit card account immediately through an Authorize.net merchant account (Online) or if the payments are to be requested later in a batch process not using Authorize.net (Offline).

For EX, the option that is selected here must also be selected in the Accounts Receivable Configuration window. If not, you will receive a warning message on the Payment Administration portlet.

+ Allow the following payment option

Select Credit Card and/or e-check. Payments by e-check (electronic check) are not allowed if payments are being authorized Offline.

+ Payment not Authorized email address (online only)

Enter the e-mail address of the person at your institution who should be notified when an unauthorized payment is received. The e-mail address is only applicable for Online authorization.

Payment Method Message

+ Message Template Name

This drop-down list is populated with message templates available for making a payment type for this portlet type. The template consists of a name and a message. The following options are available:

·       Select a previously created template from the list.

·       Modify a template by selecting it from the list and clicking the Modify button. This opens the Edit Message screen, where the template name and/or message can be edited and saved.

·       Create a new template and message by clicking the New button and entering the name and message on the Add Message screen.

·       Remove a template by selecting the appropriate template in the list and clicking the Delete button.

+ Message that will display when the Payments Selection screen appears

This message should be portlet-specific. It can only be created, modified, or deleted via the use of the Modify, New, and/or Delete buttons beside the Message Template Name drop-down. When the person making the payment in the Payments portlet clicks the Confirm and Submit button, this message is used for instructing the payor during the selection of a payment type.

Make a Payment Message

+ Message Template Name

This drop-down list is populated with message templates available for making a payment type for this portlet type. The template consists of a name and a message. The following options are available:

·       Select a previously created template from the list.

·       Modify a template by selecting it from the list and clicking the Modify button. This opens the Edit Message screen, where the template name and/or message can be edited and saved.

·       Create a new template and message by clicking the New button and entering the name and message on the Add Message screen.

·       Remove a template by selecting the appropriate template in the list and clicking the Delete button.

+ Message that will display when the Make a Payment screen appears

This message should be portlet-specific and can only be created, modified or deleted via the use of the Modify, New, and/or Delete buttons beside the Message Template Name drop-down. When the person making the payment selects the payment method, this message displays on the Enter Payment Information page of the Payments portlet to describe what has taken place.

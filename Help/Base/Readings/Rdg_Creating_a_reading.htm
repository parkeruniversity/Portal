Creating readings

Each Readings portlet instance hosts a list of titles -- suggested or required readings -- that are added by faculty members or other authorized users. Typically, readings are grouped into sets.

To create a reading:

1.        Maximize the portlet instance.

2.        Choose Manage > Add a Reading. If you do not see the Manage menu, then you do not have permission to create readings in this portlet instance.

3.        In the Title field, enter the reading’s name. This field is required.

4.        In the Author field, enter the author’s name. This field is required.

5.        If desired, enter any other necessary details in the Edition/Publisher field.

6.        If there is a supporting URL you want to include, fill out the following fields:

o      In the URL field, enter the full URL, including a protocol such as http://.

o      In the URL label field, if desired enter an informative label such as "Click here to go to university bookstore."

7.        If appropriate, enter the number of pages in the Pages field.

8.        If necessary, change the Status from the default choice of Required to any of the other choices, which are:

o      Optional

o      Suggested

o      Extra Credit

9.        Use the Set drop-down list to choose which set the reading will be part of.

10.    If appropriate, enter text in the Description field. The description can be no more than 2,000 characters. (Note that typically, this information is shown only if the user clicks through to see details about the reading -- the Description is not shown in summary views.)

11.    Click Save.

For more details on creating and managing readings and sets, refer to Jenzabar's Internet Campus Solution: Administration Guide or e-Racer: Faculty Guide.
